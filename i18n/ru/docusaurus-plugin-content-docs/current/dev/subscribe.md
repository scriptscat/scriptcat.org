---
title: Режим подписки
---

Файл должен начинаться с `UserSubscribe` вместо `UserScript`. Ссылка для установки должна использовать расширение `user.sub.js` и быть ссылкой `https`.

Скрипт подписки показывает диалог установки только для подтверждения подписки пользователем; последующие обновления выполняются тихо, и диалог обновления снова показывается только при изменении разрешения `connect`.

Один скрипт подписки может описывать ссылки установки для нескольких скриптов. Скрипты, установленные через режим подписки, устанавливаются тихо, без диалога подтверждения, и по-прежнему отображаются в списке скриптов — но их разрешение `connect` берётся из `connect`, объявленного в подписке, а не из собственного `connect` скрипта.

```js
// ==UserSubscribe==
// @name         xxx
// @description  Subscribe to the xxx script series
// @version      0.1.0
// @author       You
// @connect      www.baidu.com
// @scriptUrl    https://script.tampermonkey.net.cn/48.user.js
// @scriptUrl    https://script.tampermonkey.net.cn/49.user.js
// ==/UserSubscribe==
```

## Обновления подписки и обновления скриптов

Согласно настроенному пользователем `интервалу обновления`, ScriptCat периодически проверяет ссылку подписки; для этого должен быть задан `version`.

При каждом обновлении или изменении подписки ссылки скриптов сравниваются с уже установленными: скрипты, отсутствующие в новой подписке, удаляются, а новые устанавливаются тихо. Обновления скриптов следуют собственному `version` скрипта с той же логикой, что и при обычной установке.

## Тихая установка и обновление

Подписанные скрипты устанавливаются и обновляются тихо — добавление, удаление или обновление скрипта из подписки показывает только уведомление, без повторного подтверждения пользователем. Из‑за этого механизма тихих обновлений подписывайтесь только на доверенные источники.


## metadata

В скрипте подписки смысл некоторых полей метаданных меняется.

### name

Имя подписки (также можно редактировать прямо в списке подписок)

### description

Описание подписки — для чего она предназначена

### version

Версия подписки. Если опущена, обновления вместо этого запускаются при изменении содержимого скрипта подписки.

### connect

Запрашивает разрешение доступа к сайту; см. `GM_cookie` и `GM_xmlhttpRequest`. Для скриптов, установленных через режим подписки, `connect` переопределяется `connect` подписки.

### scriptUrl

Ссылки установки скриптов, необходимые подписке
