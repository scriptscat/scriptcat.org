## Режим разработчика / разрешения для пользовательских скриптов

#### В: ScriptCat показывает «Режим разработчика не включён», скрипты не работают?

Начиная с Chrome 120+ и новых версий Edge браузеры требуют вручную включить разрешения для запуска скриптов. См. [Включение поддержки пользовательских скриптов](/docs/use/open-dev/).

Если разрешение уже включено, а предупреждение остаётся — перезапустите браузер или перезагрузите расширение.

## Скрипты не работают

#### В: Установил(а) скрипт, но эффекта нет?

1. **Не включено «Allow User Scripts»** — см. [Включение поддержки пользовательских скриптов](/docs/use/open-dev/)
2. **Холодный старт** — сразу после открытия браузера скрипты могут ещё не загрузиться. Обновите страницу
3. **Конфликт расширений** — блокировщики рекламы (например, uBlock Origin) могут вызывать ошибки скриптов

#### В: Скрипт работает в Tampermonkey, но не в ScriptCat?

Реализации API ScriptCat и Tampermonkey немного отличаются. Обновитесь до последней версии. Если проблема остаётся — создайте Issue на [GitHub](https://github.com/scriptscat/scriptcat/issues).

## Проблемы облачной синхронизации

> Базовое использование синхронизации: [Синхронизация и резервное копирование](/docs/use/sync/).

#### В: Проблемы с синхронизацией OneDrive / Google Drive / WebDAV?

1. **Удалённые скрипты появляются снова** — включите «синхронизацию удаления» на всех устройствах

## Проблемы с авторизацией Cookie

#### В: GM_cookie не получает cookies?

1. **Не появляется окно авторизации** — убедитесь, что `GM_cookie` объявлен в `@grant` скрипта, и через `@connect` указаны нужные домены

## Потеря данных скриптов

#### В: После открытия браузера все скрипты пропали?

1. **Задержка инициализации** — при старте браузера ScriptCat ещё может загружать данные. Подождите несколько секунд или перезапустите браузер
2. **«Чистильщики»** — 360 Security Guard, CCleaner и подобные могут удалять данные расширений. Исключите данные расширений браузера в настройках очистки
3. **Регулярные бэкапы** — используйте экспорт или [облачную синхронизацию](/docs/use/sync/) для резервного копирования скриптов и настроек
